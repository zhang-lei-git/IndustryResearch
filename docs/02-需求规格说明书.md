# 区域产业调研与机会决策系统需求规格说明书

## 1. 术语与角色

| 术语 | 说明 |
| --- | --- |
| 区域工作空间 | 一个可独立开展产业调研、数据管理和决策分析的区域单元，例如“阎良区航空制造产业” |
| 产业专题 | 围绕产业链环节、核心协同、政策方向或技术主题建立的研究主题 |
| 企业档案 | 企业基础画像、调研证据、动态情报、政策适配、行动与结论的统一入口 |
| 调研假设 | 调研前对某类企业、产业环节或主题形成的待验证判断 |
| 证据 | 支撑结论的录音、纪要、外部情报、政策原文、招投标线索或附件 |
| 平台级资产 | 可跨区域复用的能力库、案例库、问题模板、行业模型和政策解析 |

首期用户为单人；系统设计需为未来调研团队、区域负责人、行业专家和管理层协作预留扩展空间。

## 2. 总体需求

### 2.1 产品定位

系统是“区域产业调研与机会决策工作台”，不能退化为静态企业展示或产品机会清单。

### 2.2 核心原则

1. 产业与客户问题优先，既有产品与服务能力匹配后置。
2. 每个关键判断尽量绑定来源、时间、可信等级和待验证状态。
3. 企业详情在任何入口展示一致，作为可复用的统一企业档案。
4. 区域内数据完整隔离，平台级知识可复用，跨区域洞察可比较。
5. 支持人工维护、自动更新和 AI 辅助三种信息获取方式。

## 3. 功能需求

### FR-01 区域工作空间管理

| 编号 | 需求 |
| --- | --- |
| FR-01-01 | 系统以区域工作空间承载企业、产业地图、政策、情报、调研和洞察。 |
| FR-01-02 | 首期内置“阎良区航空制造产业”工作空间。 |
| FR-01-03 | 后期可新增区域、设置区域名称、行政范围、产业主线、重点专题和默认政策范围。 |
| FR-01-04 | 所有区域业务数据必须具备 `workspaceId` 归属。 |
| FR-01-05 | 平台级能力库、案例库和问题模板可被多个工作空间引用。 |

### FR-02 产业研究工作台

| 编号 | 需求 |
| --- | --- |
| FR-02-01 | 支持产业分类视图：按研发设计、材料工艺、零部件、结构件、总装、试飞适航、维修保障、低空等环节展示企业。 |
| FR-02-02 | 支持核心节点协同网络：围绕一飞院、西飞、试飞院、强度所等节点展示设计、制造、验证、试飞和平台协同关系。 |
| FR-02-03 | 支持产业专题、调研假设、证据和结论的关联展示。 |
| FR-02-04 | 支持将政策、动态情报、企业调研结论作为产业地图叠加信息。 |
| FR-02-05 | 企业卡片和核心网络中的企业标签均可打开统一企业详情。 |

### FR-03 企业统一档案

| 编号 | 需求 |
| --- | --- |
| FR-03-01 | 企业详情在企业名单、产业地图、搜索结果、后续政策/情报列表等所有入口一致。 |
| FR-03-02 | 展示基础信息：名称、区域、行业、企业类型、产业链位置、规模、联系人、调研状态、数字化成熟度、说明。 |
| FR-03-03 | 展示产业调研关注点、已验证需求、反例、关键联系人、历史访谈和后续行动。 |
| FR-03-04 | 展示招投标/采购/新闻/招聘/项目建设等动态情报，包含来源、发布时间、采集时间、可信等级、关联主题和待验证状态。 |
| FR-03-05 | 展示适配政策，包含支持方向、金额/方式、适配理由、可匹配服务和使用建议。 |
| FR-03-06 | 支持统一编辑；任一入口编辑后，所有入口即时显示同一份数据。 |

### FR-04 调研计划与任务包

| 编号 | 需求 |
| --- | --- |
| FR-04-01 | 支持按企业、企业群、产业环节和专题创建调研计划。 |
| FR-04-02 | 计划应包含目标、待验证假设、参与人、时间、路线、准备材料和预期产出。 |
| FR-04-03 | 系统按企业产业角色、近期动态、政策适配、历史记录和问题模板生成调研问题包。 |
| FR-04-04 | 调研问题应支持分类、编辑、保存、复用和版本记录。 |
| FR-04-05 | 支持将问题与调研假设、信息源和预期决策关联。 |

### FR-05 调研记录与证据沉淀

| 编号 | 需求 |
| --- | --- |
| FR-05-01 | 支持录音上传；后续支持录音转写和重点片段定位。 |
| FR-05-02 | 每次记录至少关联企业、日期、调研人员、纪要、原始证据、需求、结论与下一步动作。 |
| FR-05-03 | 支持从纪要中提取问题、需求、预算/项目窗口、政策诉求、招标线索和风险项。 |
| FR-05-04 | 支持标注“已验证、待验证、反例、不确定”状态。 |
| FR-05-05 | 支持按产业环节、企业类型、主题和时间检索记录。 |

### FR-06 动态情报与更新

| 编号 | 需求 |
| --- | --- |
| FR-06-01 | 支持人工新增和编辑外部情报。 |
| FR-06-02 | 后续支持按企业名、产业关键词、数字化关键词和政策关键词配置采集任务。 |
| FR-06-03 | 情报包括招投标、采购、政策、新闻、招聘、扩产、认证和活动等类型。 |
| FR-06-04 | 每条情报需保存来源 URL、原文摘要、发布时间、采集时间、关联企业/专题、可信等级和是否需要验证。 |
| FR-06-05 | 系统可根据新情报推荐调研问题或创建待办，但不能直接将线索等同于需求结论。 |

### FR-07 政策研究与政策杠杆

| 编号 | 需求 |
| --- | --- |
| FR-07-01 | 支持维护国家、省、市、区/园区多级政策。 |
| FR-07-02 | 政策需标注适用产业、企业条件、支持方式、申报窗口、数字化关联方向和来源。 |
| FR-07-03 | 系统按企业类型、产业位置、发展阶段和调研证据推荐政策适配结果。 |
| FR-07-04 | 政策结果需说明“为什么匹配”和“能如何降低客户立项/投入门槛”。 |
| FR-07-05 | 支持从产业专题角度分析政策引导方向与服务能力空白。 |

### FR-08 洞察与决策支持

| 编号 | 需求 |
| --- | --- |
| FR-08-01 | 汇总企业需求，输出按产业环节、企业类型、主题、时间的共性需求与反例。 |
| FR-08-02 | 形成“待验证假设—证据—结论—建议”的可追溯链路。 |
| FR-08-03 | 形成市场进入、销售策略、行业方案研发、生态合作和政策包装五类建议。 |
| FR-08-04 | 支持生成区域月度/阶段性研究简报。 |
| FR-08-05 | 后续支持跨区域比较：相同产业角色的需求热度、政策环境、项目成熟度和进入策略。 |

## 4. 数据需求

### 4.1 核心实体

- 区域工作空间
- 产业模型与产业环节
- 企业与集团关联
- 企业关系与核心协同
- 调研计划、任务、问题包
- 调研记录、录音、转写、纪要、需求、行动项
- 外部情报与采集任务
- 政策与政策适配
- 调研假设、证据、专题、洞察、决策建议
- 平台级能力、案例、方案资产和问题模板

### 4.2 数据质量要求

每条外部信息和重要结论应至少包含：来源、记录/采集时间、归属区域、关联对象、状态。对推断性内容需要标记“待验证”，避免把公开线索写成既成事实。

## 5. 非功能需求

| 类别 | 要求 |
| --- | --- |
| 易用性 | PC 为主要工作界面，手机端适合查看企业档案、记录访谈和处理待办。 |
| 性能 | 首期单区域数百企业、数千条记录时应保持常规操作流畅。 |
| 可扩展性 | 数据模型和 API 预留区域工作空间字段与平台级资产域。 |
| 可维护性 | 业务模块、数据模型、采集任务和 AI 规则应可独立演进。 |
| 安全性 | 后续接入登录后，应支持区域级数据范围、角色权限、敏感附件访问控制和操作审计。 |
| 合规性 | 公开信息采集遵循来源规则；访谈录音和企业信息需明确使用范围与保存策略。 |

## 6. 验收标准

1. 用户可从任意企业入口打开一致的企业详情抽屉。
2. 用户可查看企业基础信息、调研记录、招投标/情报和政策适配。
3. 用户可按企业产业角色生成并编辑调研问题包。
4. 用户可完成计划、记录、需求和结论的基本闭环。
5. 用户可从产业地图观察企业分布和核心节点协同。
6. 系统数据模型能够在不迁移业务语义的前提下新增第二个区域工作空间。
